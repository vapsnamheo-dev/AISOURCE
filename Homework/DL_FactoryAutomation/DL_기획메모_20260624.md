## 1. 데이터 계획 (3-Source 통합 아키텍처)

### 원천 데이터

| 소스 | 출처 | 종류 | 규모 | 모델 |
|------|------|------|------|------|
| **Source 1** (ML) | UCI AI4I 2020 | 정형 스냅샷 | 10,000행 | XGBoost |
| **Source 2** (DL) | AI4I 물리규칙 자체 생성 | 시계열 시퀀스 | 60,000 시퀀스¹ | CNN+LSTM |
| **Source 3** (DL) | NASA UC Berkeley Milling²⁾ | 다채널 원시 신호 | 167회 절삭 실험 | 멀티채널 1D-CNN |

> ¹ 데이터2(v2): 7.5% 가우시안 노이즈 포함, train 48,000 / test 12,000 / demo 500  
> ² NASA Open Data Portal에서 직접 다운로드 (로그인 불필요, ~40MB)  
> **[주의]** PHM 2010 원본(phmsociety.org)은 원시 파형 50kHz × 수십만 샘플/파일로 용량이 크고(~2GB) 전처리가 복잡하며 회원가입이 필요합니다. 본 과제는 동일 구조의 NASA UC Berkeley Milling 데이터(~40MB, 무료 공개)로 대체 진행합니다.[^1]

[^1]: PHM 2010과 NASA UC Berkeley Milling은 동일한 실험 환경(고속 CNC 밀링, VB 플랭크 마모 측정)에서 수집된 데이터로, 피처 구조와 레이블 정의가 유사하여 대체 사용에 학술적 타당성이 있습니다.

### Source 2 DL 데이터 설계 근거
- AI4I 2020은 개별 스냅샷이라 시계열 패턴 학습 불가 → 동일 물리 규칙으로 시계열 재생성
- 시퀀스 1건 = 50타임스텝 × 9피처 (ML과 동일 피처)
- 처음 35스텝: 정상 운전, 마지막 15스텝: 해당 고장 패턴 점진 주입
- 분할: train 48,000 (80%) / test 12,000 (20%) / demo 500 (별도 시드, 미포함)
- 교차검증: Stratified K-Fold **CV=3** (클래스 비율 유지)

### Source 3 NASA Milling 데이터 설계
- 167회 절삭 실험 × 6채널 신호 (Fx/Fy/Fz 절삭력 + Vib_table + Vib_spindle + AE_spindle)
- 마모 단계 레이블: VB < 0.2mm (정상), 0.2~0.4mm (경미), > 0.4mm (심각)
- 슬라이딩 윈도우: 512 샘플 (10kHz 기준 51.2ms)
- 예상 세그먼트: ~15,000개

## 2. 전처리 파이프라인

| 단계 | Source 1 (ML) | Source 2 (DL 시계열) | Source 3 (DL 원시신호) |
|------|--------------|--------------------|--------------------|
| 결측·이상값 | dropna + clip | clip(물리범위) | clip + 선형보간 |
| 인코딩 | One-hot (Type) | Label (0/1/2) | N/A |
| 스케일링 | StandardScaler | per-feature 표준화 | per-channel min-max |
| 윈도우 구성 | N/A (스냅샷) | 50 타임스텝 슬라이딩 | 512 샘플 슬라이딩 |
| 불균형 처리 | scale_pos_weight | class_weight | class_weight |
| 저장 형식 | CSV → pkl | NPZ (48K×50×9) | NPZ (15K×512×6) |

## 7. 향후 LLM 연계 계획 (Phase 3~4)

**Phase 3 (2026 Q4) — LLM 연계**
- **RAG(검색증강생성)**: 정비 매뉴얼·과거 조치 이력을 벡터DB 색인
- **멀티모달 융합**: 센서(수치) + 이미지(설비 외관) + 문서(LLM) 통합 진단
- Gemini/Claude API 연동

**Phase 4 (2027 Q1) — 실시간 IoT 연계**
- MQTT 스트림 실시간 추론 → LLM 자연어 작업지시서 자동 생성

### 기대 효과
센서(수치) → DL(패턴) → LLM(해석·지시) 3-tier 융합으로  
**"고장 여부 감지" → "원인 진단 + 조치 가이드 자동화"** 로 진화
