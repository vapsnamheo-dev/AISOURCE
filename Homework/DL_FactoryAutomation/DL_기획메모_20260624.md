# DL_FactoryAutomation — 딥러닝 과제 기획 메모

**과제명**: AI 기반 설비 고장 예측 — ML(XGBoost) → DL(1D-CNN/LSTM) 확장
**작성일**: 2026-06-24    **담당**: 허남

---

## 1. 데이터 계획

### 원천 데이터
| 구분 | 출처 | 종류 | 규모 |
|------|------|------|------|
| ML 기반 (참조) | UCI AI4I 2020 (CC BY 4.0) | 표 형식 스냅샷 | 10,000행 |
| **DL용 합성 생성** | AI4I 물리규칙 자체 생성 | **시계열 시퀀스** | **6,200 시퀀스** |

### DL 데이터 설계 근거
- AI4I 2020은 개별 스냅샷이라 시계열 패턴 학습 불가 → 동일 물리 규칙으로 시계열 재생성
- 시퀀스 1건 = 50타임스텝 × 9피처 (ML과 동일 피처)
- 처음 35스텝: 정상 운전, 마지막 15스텝: 해당 고장 패턴 점진 주입
- 분할: train 4,800 (80%) / test 1,200 (20%) / demo 200 (별도 시드, 미포함)
- 교차검증: Stratified K-Fold **CV=3** (클래스 비율 유지)

### 피처 (ML과 동일 9종)
air_temp_k, process_temp_k, rotational_speed_rpm, torque_nm, tool_wear_min,
type_encoded(L=0/M=1/H=2), power_w(파생), overstrain_minnm(파생), temp_diff_k(파생)

## 2. 전처리 파이프라인

| 단계 | 방법 | ML 대비 차이점 |
|------|------|--------------|
| 결측·이상값 | dropna + clip(물리범위) | 동일 |
| 범주 인코딩 | Type → 0/1/2 (Label) | 동일 |
| 스케일링 | **train 기준 표준화 (mean/std)** | ML: StandardScaler → DL: per-feature 표준화 |
| 시퀀스 구성 | 50 타임스텝 슬라이딩 윈도우 | **NEW — ML에 없음** |
| 불균형 처리 | class_weight (고장 비율 역수) | ML: scale_pos_weight → DL: class_weight |
| 저장 형식 | numpy NPZ 압축 (6.9MB + 1.7MB) | ML: CSV → DL: NPZ (46MB → 8.6MB) |

## 3. 비교 모델 (3종)

| 모델 | 구조 | 특징 | 학습 파라미터 (예상) |
|------|------|------|-------------------|
| **1D-CNN** | Conv1D×3 → GlobalAvgPool → Dense | 단기 이상 패턴 포착 (3~7스텝 커널) | ~120K |
| **LSTM** | LSTM(128) → LSTM(64) → Dense | 장기 의존성 (누적 열화 추세) 학습 | ~180K |
| **CNN+LSTM** ⭐ | Conv1D×2 → LSTM(128) → LSTM(64) → Dense | 단기+장기 동시 포착 (권장) | ~200K |

모두 이진분류(고장/정상)로 학습. 향후 다중분류(고장유형 6종) 확장 예정.

평가 지표: **Recall ≥ 82%, ROC-AUC ≥ 0.97, F1** (고장 미검출 최소화 우선)

## 6. 향후 LLM 연계 계획 (Phase 3~4)

### 단계별 로드맵

**Phase 2 (2026 Q3) — 현재 DL 고도화**
- 실측 공장 데이터 연동 (PostgreSQL 영구 저장)
- SHAP 해설 (어느 타임스텝에서 고장 신호 발생했는지 시각화)
- 다중분류 확장 (TWF/HDF/PWF/OSF/RNF 유형 구분)

**Phase 3 (2026 Q4) — LLM 연계**
- **RAG(검색증강생성)**: 정비 매뉴얼·과거 조치 이력을 벡터DB 색인
  - 예측된 고장 유형 → 관련 매뉴얼 검색 → 조치 방법 자동 제안
- **멀티모달 융합**: 센서(수치) + 이미지(설비 외관) + 문서(LLM) 통합 진단
- Gemini/Claude API 연동, API 없어도 규칙 기반 폴백 동작 유지

**Phase 4 (2027 Q1) — 실시간 IoT 연계**
- MQTT 스트림으로 실시간 시퀀스 수집 → DL 모델 즉시 추론
- 경보 발령 → LLM이 자연어 작업지시서 자동 생성
- 모바일 반응형 대시보드

### 기대 효과
센서(수치) → DL(패턴) → LLM(해석·지시) 3-tier 융합으로
**"고장 여부 감지" → "원인 진단 + 조치 가이드 자동화"** 로 진화
