# CNC 설비 고장 예측 - 3-Source DL 통합 상세 보고서

작성일: 2026-06-24  담당: 허남

---

## 1. 프로젝트 개요

### 1.1 목표
ML(머신러닝) 기반 설비 고장 예측 시스템을 DL(딥러닝)으로 확장하여
처리 가능한 데이터 유형과 탐지 정확도를 개선합니다.

### 1.2 핵심 주장
"DL은 ML이 처리하지 못하는 시계열/원시신호 데이터를 추가로 활용하여
고장 탐지 범위와 정확도를 함께 개선한다"

## 2. 데이터 설계

### Source 1: AI4I 2020 (ML 기준선)
- 출처: UCI ML Repository (CC BY 4.0)
- 규모: 10,000행, 14개 원시피처 -> 11개 처리 피처
- 고장 비율: 3.4% (불균형 심각)
- 분할: 80:20 train/test, CV=5

### Source 2: 합성 시계열 (DL 시계열)
- 생성 방법: AI4I 물리 규칙 기반 자체 생성 (Python)
- 규모: 60,000 시퀀스 x 50타임스텝 x 9피처
- 데이터1(v1): 6,000 시퀀스, 노이즈 없음 -> Test 100% (이상적)
- 데이터2(v2): 60,000 시퀀스, 가우시안 7.5% 노이즈 -> 현실적 성능
- 고장 비율: 35% (균형 설계)
- 분할: 80:20 + CV=3

### Source 3: NASA UC Berkeley Milling
- 출처: NASA Open Data Portal (공개, 무료)
- 규모: 167회 절삭 실험 x 9,000샘플 x 6채널
- 채널: smcAC, smcDC, vib_table, vib_spindle, AE_table, AE_spindle
- 레이블: VB 플랭크 마모 측정값 -> 이진(정상:VB<0.2mm / 마모:VB>=0.2mm)
- 전처리: 슬라이딩 윈도우 512샘플/스텝128 -> 9,782 세그먼트
- 분할: 80:20 stratified

## 3. 모델 아키텍처

### Source 2 - CNN+LSTM (시계열)
  입력: (50, 9)
  Conv1D(64,3) -> BN -> MaxPool(2) -> Conv1D(128,3) -> BN -> MaxPool(2)
  -> LSTM(128, return_sequences=True) -> LSTM(64)
  -> Dropout(0.3) -> Dense(64) -> Dense(1, sigmoid)
  파라미터: ~200K

### Source 3 - 멀티채널 1D-CNN (원시신호)
  입력: (512, 6)
  Conv1D(64,7) -> BN -> MaxPool(2) -> Conv1D(128,5) -> BN -> MaxPool(2)
  -> Conv1D(256,3) -> BN -> GlobalAvgPool
  -> Dense(128) -> Dropout(0.3) -> Dense(1, sigmoid)
  파라미터: ~150K

## 5. ML vs DL 비교 분석

### 5.1 처리 가능 데이터 유형

| 데이터 유형 | ML | DL | 비고 |
|-----------|----|----|------|
| 정형 테이블 | O | O | ML이 더 빠름 |
| 시계열 패턴 | 제한적 | O | DL 압도적 우위 |
| 원시 진동신호 | FFT 전처리 필수 | O | DL이 자동 추출 |
| 이미지 | 불가 | O (CNN) | 향후 확장 가능 |

### 5.2 Feature Engineering 비교

ML 파이프라인:
  원시신호 -> FFT 계산 -> 통계 피처 추출 -> 정규화 -> 학습
  (도메인 전문가 + 수작업 필요)

DL 파이프라인:
  원시신호 -> Conv1D 자동 피처 추출 -> 학습
  (end-to-end, 자동화)
