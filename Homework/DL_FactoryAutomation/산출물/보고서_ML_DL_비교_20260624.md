## 3-계층 아키텍처

  [Layer 1] SCADA 정형 데이터 -> ML (XGBoost/RF)
    - 설치 용이, 기존 시스템 연동, 빠른 추론
    - AI4I 기준: F1=0.724, ROC-AUC=0.971

  [Layer 2] 시계열 센서 추이 -> DL (CNN+LSTM)
    - ML이 탐지 못하는 시간적 패턴 포착
    - 합성 v1: F1=1.000 / v2 (현실적): 학습 중

  [Layer 3] 진동/음향 원시신호 -> DL (1D-CNN)
    - 피처 엔지니어링 없이 end-to-end 학습
    - 현황: 전처리 개선 필요 (1차 시도 ROC-AUC=0.50)

## Source 3 실패 분석 (학습 포인트)

RF와 1D-CNN 모두 동일한 실패:
- 혼동행렬 [[670, 0], [1287, 0]] - Worn을 전혀 탐지 못함
- 원인: 슬라이딩 윈도우 레이블 할당 또는 채널 정규화 문제
- 교훈: DL도 데이터 전처리 품질이 학습 성패를 결정함
- 대응: VB 임계값 재검토 + SMOTE + 절삭 조건별 정규화 (Phase 2.5)

## 공장 실무 적용 가능성

Siemens MindSphere, GE Predix, Bosch AI Factory 등 실제 Production PdM 시스템은
동일한 3-계층 구조를 사용합니다:

- Layer 1 (ML): 기존 SCADA 연동 - 즉시 배포 가능
- Layer 2 (DL/시계열): 열화 추이 포착 - IoT 센서 필요
- Layer 3 (DL/원시신호): 고주파 진동/음향 - 전용 센서 추가 필요

각 계층이 이전 계층이 탐지 못하는 것을 포착하므로
전체 시스템 탐지 범위(Recall)가 확장됩니다. -> Reasonable & Makes Sense
