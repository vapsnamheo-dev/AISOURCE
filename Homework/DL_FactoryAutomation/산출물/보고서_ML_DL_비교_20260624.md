## Q6 결론: ML vs DL 직접 비교

### Source 3 (NASA Milling 동일 데이터)

RF (ML):    Accuracy=0.342  F1=0.000  ROC-AUC=0.500  → 전체 Normal 예측
1D-CNN (DL): Accuracy=0.658  F1=0.793  ROC-AUC=0.500  → 전체 Worn 예측

→ 두 방법 모두 ROC-AUC=0.500 (랜덤 수준). 실질 학습 미발생.

### 데이터셋 한계 (근본 원인)

1. 실질 독립 샘플 146개 (절삭 단위). 슬라이딩 윈도우 9,782개는 증강이지 추가 정보가 아님.
2. 절삭 조건(이송속도/절삭깊이) 변화가 마모 신호보다 큰 노이즈로 작용.
3. NaN 채널 존재 → 전처리 오염 → 모든 피처 NaN.

### DL > ML 인가?

이번 실험: 동등하게 실패 (AUC=0.500 기준).
문헌(Lee et al. 2021): 올바른 전처리 하에서 1D-CNN AUC 0.91 vs RF 0.78.
결론: 전처리 완성도가 모델 선택보다 더 중요.

## Insight

### Insight 1: 데이터 품질 > 모델 선택
Source 3에서 RF와 1D-CNN이 동일하게 실패.
전처리 실패 시 어떤 모델도 학습하지 못한다.

### Insight 2: 독립 샘플 수가 결정적
146회 절삭이 실질 샘플 수. 슬라이딩 윈도우는 정보를 늘리지 않는다.
DL과 ML 모두 데이터 부족 앞에서 동일하게 실패한다.

### Insight 3: DL의 강점 = 피처 자동 추출 (충분한 데이터 전제)
Source 2 v2에서 CNN+LSTM F1=1.000 vs XGBoost F1=0.724.
시계열 패턴을 수동 피처 없이 자동 포착하는 것이 DL의 실질 강점.

### Insight 4: 3-계층 구조 타당성 재확인
L1 (ML/정형): 즉시 적용 가능, ROC-AUC=0.97
L2 (DL/시계열): 합성 검증 완료, 실제 데이터 수집 후 3-6개월 내 적용 가능
L3 (DL/원시신호): 실제 데이터에서 전처리 난이도 확인 → 6-12개월 준비 필요
