## 2. 전처리 상세

### 2.1 피처 추출 (스냅샷별)

각 스냅샷(100ms, 2560 샘플)에서 시간영역 통계 피처를 추출:

| 피처 | 수식 | 의미 |
|------|------|------|
| h_rms | √(mean(x²)) | 수평 진동 에너지 |
| h_kurt | E[(x-μ)⁴]/σ⁴ | 수평 충격성 (정규 = 3) |
| h_skew | E[(x-μ)³]/σ³ | 수평 비대칭성 |
| h_crest | peak / rms | 수평 파고율 |
| v_rms, v_kurt, v_skew, v_crest | (수직 채널 동일) | 수직 방향 |
| temp_mean | mean(온도) | 베어링 온도 평균 |

**이은주 ML 방식 피처와 대응**:
- 이은주: h_rms, h_kurt, h_skew, h_crest (수평 4개)
- 본 프로젝트: 수평 4 + 수직 4 + 온도 1 = **9개** (확장)

### 2.2 열화 라벨 생성 (이은주 방식 적용)

```python
# 각 베어링 초기 10분 h_rms 평균
init_10_rms = df["h_rms"].iloc[:10].mean()

# 이은주 임계값: 초기 평균 × 2.5
threshold = init_10_rms * 2.5

# 열화 라벨
df["label"] = (df["h_rms"] > threshold).astype(int)
```

**근거**: 이은주 논문에서 검증된 임계값 배수(2.5×)로 초기 정상 상태 대비 유의미한 열화 시점 식별.

### 2.3 VIF 분석 결과

다중공선성 진단으로 과잉 피처 제거:

| 피처 | VIF | 판정 |
|------|-----|------|
| h_rms | [RESULT] | [양호/주의/심각] |
| h_kurt | [RESULT] | [양호/주의/심각] |
| h_skew | [RESULT] | [양호/주의/심각] |
| h_crest | [RESULT] | [양호/주의/심각] |
| v_rms | [RESULT] | [양호/주의/심각] |
| v_kurt | [RESULT] | [양호/주의/심각] |
| v_skew | [RESULT] | [양호/주의/심각] |
| v_crest | [RESULT] | [양호/주의/심각] |
| temp_mean | [RESULT] | [양호/주의/심각] |

**VIF 기준**: 양호(<5), 주의(5~10), 심각(≥10 → 제거)

### 2.4 RUL 계산

```python
# 분 단위 RUL (시험 중단 시점까지의 잔여 분)
total_minutes = len(df_bearing)
df["rul"] = total_minutes - df["minute"]
df["rul_pct"] = df["rul"] / total_minutes  # 0~1 정규화
```

### 2.5 Demo Mode (합성 데이터)

실제 FEMTO 데이터 없을 시 자동으로 합성 데이터 생성:

```
열화 곡선: rms = baseline × (1 + 5 × max(0, progress - 0.7)²)
- 초반 70%: 거의 평탄 (정상 운전)
- 후반 30%: 급격한 RMS 증가 (가속 열화)
```

## 3. ML 방법론

### 3.1 이은주+사용자 ML 프로젝트 결합 방식

| 요소 | 이은주 방식 | 사용자 ML 프로젝트 | 본 FEMTO 프로젝트 |
|------|------------|-------------------|-----------------|
| 피처 | h_rms 등 4개 | 9개 (공정+파생) | h/v 8개 + 온도 = 9개 |
| VIF 분석 | 미수행 | statsmodels VIF | statsmodels VIF (동일) |
| 모델 | RF 위주 | LR/RF/XGB 3종 | LR/RF/XGB 3종 |
| 교차검증 | GroupKFold | Train/Test 분할 | GroupKFold (베어링 단위) |
| GridSearch | 미수행 | 수행 | 수행 |
| 라벨 기준 | h_rms × 2.5 | 고장 플래그 | h_rms × 2.5 (이은주 방식) |

### 3.2 GroupKFold 적용 이유

시계열 데이터에서 랜덤 분할 시 미래 데이터가 학습에 유입됨(누수).  
베어링 단위 그룹 분할로 테스트 베어링은 학습에서 완전 분리.

```
GroupKFold(n_splits=3):
  Fold 1: Train=[Bearing1_2, Bearing1_3] / Val=[Bearing1_1]
  Fold 2: Train=[Bearing1_1, Bearing1_3] / Val=[Bearing1_2]
  Fold 3: Train=[Bearing1_1, Bearing1_2] / Val=[Bearing1_3]
```

## 4. DL 방법론

### 4.1 LSTM 아키텍처

```
Input: (batch, 30, n_features)  ← 30분 슬라이딩 윈도우
  ↓
LSTM(64, return_sequences=True)
  ↓
Dropout(0.2)                    ← 과적합 방지
  ↓
LSTM(32)
  ↓
Dense(16, activation='relu')
  ↓
Dense(1)                        ← RUL (분 단위, 연속값)
```

**손실 함수**: MSE (회귀)  
**옵티마이저**: Adam (lr=1e-3)  
**EarlyStopping**: patience=5, val_loss 기준

### 4.2 슬라이딩 윈도우

```
윈도우 = 30분 시퀀스 → 다음 시점 RUL 예측

t=1~30  → RUL at t=31
t=2~31  → RUL at t=32
...
```

**30분 선택 이유**: 열화 패턴이 단기(수 분)~중기(수십 분)에서 형성되며, 너무 짧으면 패턴 미학습, 너무 길면 초기 정상 데이터가 희석됨.

### 4.3 RF 베이스라인 비교

LSTM의 실질적 기여를 측정하기 위해 단순 RF 회귀(마지막 타임스텝 피처)와 비교:

- **RF 베이스라인**: 시계열 의존성 없이 현재 상태만으로 RUL 예측
- **LSTM**: 30분 추세를 반영하여 RUL 예측
- **개선률**: (RF_RMSE - LSTM_RMSE) / RF_RMSE × 100%

## 7. 임계값 UI 설계 의도

### 7.1 ML 열화 판정 임계값

기존 ML 프로젝트(ML_FactoryAutomation)에서 동일한 UI 패턴 이식:

**설계 의도**:
- 설비 중요도/위험도에 따라 운영자가 실시간 조정
- 0.3 이하: 민감 모드 (안전 설비, 놓침 최소화)
- 0.5 기본: 균형 모드
- 0.7 이상: 보수 모드 (오경보 최소화)

### 7.2 DL RUL 경보 임계값 (신규)

**설계 의도**:
- 부품 조달 리드타임에 맞춰 설정
- 예: 부품 납기 3일 = 약 4,320분 → 여유 있게 300분으로 설정
- 교대 근무 고려: 60~120분 권장
